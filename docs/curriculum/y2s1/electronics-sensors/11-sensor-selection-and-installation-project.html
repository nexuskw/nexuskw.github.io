01 · Define the measurement first

Before any sensor catalog is opened, the actual measurement needs must be pinned down: what physical quantity, over what range, at what accuracy the application genuinely requires. Over-specifying — demanding laboratory-grade accuracy for a rough trend indication — wastes money on unnecessary precision; under-specifying risks a control or safety decision built on a measurement that cannot actually resolve what it needs to. This single step, done carelessly, is the root cause of a large share of "wrong sensor" purchases — not a failure of the electronics that came later.

02 · Match the signal and wiring convention

With the measurement defined, the sensor's output must match the receiving system: an analog quantity typically means a 4-20 mA loop (Lesson 5) with its live-zero fault detection; a yes/no state means a discrete output, correctly matched for sourcing or sinking against the input card it will connect to (Lesson 4) — checked before purchase, not discovered during commissioning. A small, inherently noisy signal (a thermocouple, a strain-gauge bridge) additionally needs an instrumentation amplifier with adequate CMRR for the expected noise environment (Lesson 3).

03 · Weigh the installation constraints

The sensor also has to survive and perform in its actual physical installation: mounting method, chosen for the required accuracy and (for vibration) frequency range against cost and convenience (Lesson 8); protection against the process environment, weighed against the response-lag cost that protection can add (thermowells, Lesson 6); and the electrical noise environment, addressed through shielding, twisted-pair routing, correct single-point grounding, and — for genuinely difficult cases — galvanic isolation (Lesson 10). None of these are optional footnotes; each is a real, previously-demonstrated failure mode when skipped.

04 · Document the decision — the specification sheet

The output of this process is a documented specification: measured quantity and range, required accuracy, output signal type and wiring convention, mounting method, environmental protection, and any special wiring or grounding requirements — nothing left to be discovered later by whoever happens to install it. A worked example ties the whole course together: specifying a bearing-temperature point on rotating equipment means working through all four steps — range and accuracy from the equipment's known operating temperature (1); an RTD, not a thermocouple, chosen for its superior accuracy and stability at that moderate temperature, output as 4-20 mA (2, Lessons 5-6); a thermowell for maintainability, its response lag checked against how fast the process can actually change (3, Lesson 6); and a documented spec sheet recording all of it, including the low RTD excitation current chosen to keep self-heating error negligible (4) — the same reasoning this course has built lesson by lesson, now applied as one deliberate sequence rather than an afterthought.

Key discipline — the specification process

Define the measurement → match signal and wiring → weigh installation constraints → document the decision. Skipping a step does not remove the question, it only delays discovering the answer.

Problem 1 of 11 · solve, then check

Full specification walkthrough

Problem. A pump bearing needs continuous temperature monitoring in the 60-100°C range for a condition-monitoring program valuing long-term accuracy and stability, feeding an existing 4-20 mA-based control system. Walk through the four-step specification process and state the key decision at each step.

Solution. (1) Define: 60-100°C range, high accuracy/stability required for trending over years — this rules out a coarse-accuracy option. (2) Match: an RTD (Lesson 6) fits better than a thermocouple at this moderate range given the accuracy/stability requirement; output as 4-20 mA to match the existing system (Lesson 5), with a live-zero fault check built in for free. (3) Install: a thermowell protects the sensor and allows replacement without shutting down the pump, accepting the resulting response lag since bearing temperature changes slowly (Lesson 6); low RTD excitation current is specified to keep self-heating error negligible. (4) Document: all of the above recorded in a spec sheet — range, accuracy, sensor type, signal type, thermowell, and excitation current — before ordering.

Problem 2 of 11 · solve, then check

Spotting a skipped step

Problem. A new discrete proximity sensor is purchased and installed based only on its detection range and price, with no check against the PLC input card’s sourcing/sinking configuration. On commissioning, it does not work. Which step of the specification process was skipped, and what should have caught the problem earlier?

Solution. Step 2 (match signal and wiring) was skipped — detection range and price alone address only step 1 (define the measurement). Checking the sourcing/sinking convention of both the sensor and the PLC input card before purchase, as step 2 requires, would have caught the mismatch on paper, before any installation labor was spent discovering it in the field.

Problem 3 of 11 · solve, then check

Over-specification has a cost too

Problem. A tank level only needs to be monitored as a rough trend indicator, with no control action tied to it. Explain, using 1, why specifying a high-accuracy, temperature-compensated DP transmitter with a wet-leg installation would likely be the wrong choice here, even though it would technically work.

Solution. Step 1 requires matching accuracy to actual need, not maximizing it. A rough trend indicator does not need — and should not pay for — a high-accuracy, temperature-compensated transmitter and the added installation complexity of a wet leg; a simpler, lower-cost level sensor adequate for trending would satisfy the actual requirement at a fraction of the cost, and the elaborate solution's accuracy would be wasted on a need that never required it.

Applied case — factory floor (Kuwait)

An instrumentation engineer reviewing a new project's sensor list before it goes out for purchase — checking each line against all four steps, not just price and delivery time — is doing exactly the work this lesson describes, and is the reason a well-run project's commissioning phase is comparatively uneventful: nearly every "surprise" wiring or compatibility problem on a poorly-run project traces back to one of these four questions never having been asked on paper.

A maintenance department's sensor failure log, reviewed periodically for patterns — the same wiring-convention mismatch recurring across different projects, the same mounting method failing to catch the same class of vibration fault — is using field experience to feed back into step 1 and step 3 for the next specification, turning this lesson's process from a one-time checklist into a genuinely improving standard over time.
