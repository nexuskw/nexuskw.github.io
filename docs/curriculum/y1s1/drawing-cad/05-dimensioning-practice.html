Dimensioning practice

Datum thinking: dimension from function, not convenience. Chain versus baseline dimensioning and tolerance stack-up. Over-dimensioning as an error, not a courtesy.

Lecture

After this lesson you can answer:
  • Chain versus baseline dimensioning: which accumulates tolerance, and why?
  • What makes an over-dimensioned drawing an error rather than extra information?

01 · Dimensions carry function

A dimension is not decoration; it encodes which distances matter and from where. Good practice dimensions each feature once, from datums that mean something to the part's function, placed in the view where the feature shows true shape. Duplicated or over-complete dimensioning is an error, not generosity: two paths to the same edge will one day disagree, and the shop must guess which one you meant.

02 · Chains, baselines, and the accumulation problem

Chain dimensioning strings features end-to-end: each step is controlled, but the position of the last feature accumulates every tolerance in the chain — three steps at ±0.1 can put the final hole ±0.3 from the first. Baseline dimensioning measures every feature from one datum: each position is held directly, and accumulation dies. The choice is functional: chain when step-to-step spacing matters (mating hole pairs), baseline when positions from a reference matter (mounting patterns).

03 · The grammar of callouts

⌀ before a value means diameter; R radius; ( ) marks a reference dimension (information, not control); a boxed dimension is theoretically exact — controlled elsewhere, usually by GD&T. Units are declared once in the title block; mixed units on a face of a drawing are a defect. Reading a dimension means reading its status: controlled, reference, or basic — three different levels of promise.

Closing. Dimension once, from functional datums, chain or baseline by intent, and read every callout's status before trusting it. The dimensions are the part's legal description; everything else is scenery.

Foundations

What this lesson assumes

  • Views and sections (Lessons 2, 4) — dimensions live where shape shows true.
  • Tolerance intuition — every dimension implies a permitted variation (Lesson 6 formalizes).

Glossary

  • Datum — the reference a dimension originates from.
  • Chain dimensioning — feature-to-feature; accumulates tolerance.
  • Baseline dimensioning — all from one datum; kills accumulation.
  • Reference dimension — parenthesized; informative, not binding.
  • Basic dimension — boxed; exact by definition, toleranced through GD&T.

Examples and Quiz

Worked example 1 — accumulation, quantified

Problem. Four holes in a strip are chain-dimensioned 50±0.1, 50±0.1, 50±0.1 from hole 1. Find: the worst-case position error of hole 4 relative to hole 1, and the baseline alternative.

Solution. (1) Chain: errors add — worst case \( \pm(0.1+0.1+0.1) = \pm0.3 \) mm. (2) Baseline: dimension holes at 50±0.1, 100±0.1, 150±0.1 from hole 1 — each position now held to ±0.1 directly. (3) Note the trade: baseline loosens control of spacing between adjacent holes (two ±0.1 positions → spacing ±0.2), so the functional question — do positions or spacings matter? — decides the scheme.

Answer: chain worst case ±0.3; baseline holds each position to ±0.1 at the cost of ±0.2 on adjacent spacing. Sanity check: tolerances add linearly in worst case — three dice, three chances to land long.

Worked example 2 — the double-dimension trap

Problem. A plate 120 mm long carries a slot dimensioned 40 from the left edge and 80 from the right edge, both toleranced ±0.2. Find: the defect.

Solution. (1) The dimensions over-define the slot: 40 + 80 = 120 only if the plate is exactly 120. (2) With the plate itself at 120±0.3, a shop holding both slot dimensions is being asked to control the plate length through the slot — impossible as written. (3) Fix: one dimension governs (say 40±0.2 from the functional edge); the other becomes reference (80) or is deleted.

Answer: over-dimensioning; demote one dimension to reference. Sanity check: ask "could both be simultaneously out?" If yes, the scheme fights itself.

Applied case — factory floor (Kuwait)

Vignette — the base that wouldn't bolt down. You arrive at a motor swap where the new base's holes miss the foundation bolts by 2 mm on the far pair — every hole individually "in tolerance" on the shop's report. The drawing chained the pattern; the tolerances accumulated exactly as Lesson-5 arithmetic predicts, and the far holes wandered. The concrete wins arguments with steel, so the base gets slotted this time — and the drawing gets a baseline-dimensioned revision so the next base arrives right. You attach worked example 1 to the revision request verbatim; document control approves it same-day.
