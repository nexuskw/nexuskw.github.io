Parametric CAD: parts

Sketch–constrain–feature workflow in any parametric CAD tool. Design intent: modeling so the part edits sanely. Fully defined sketches as a discipline.

Lecture

After this lesson you can answer:
  • What is design intent in a parametric model, and how does a bad sketch betray it?
  • Why should sketches be fully defined before features are built?

01 · Parametric means rebuildable

Parametric CAD stores recipes, not shapes: a sketch with dimensions, then features — extrude, revolve, hole, fillet — stacked in a history tree. Change a dimension and the recipe re-executes. The power is enormous and conditional: a model rebuilt from a disciplined recipe updates cleanly; a model built on accidental geometry collapses into red errors the first time anyone touches it.

02 · Design intent — the recipe's grammar

Design intent is deciding what should stay true when things change. A hole 10 mm from the edge (offset intent) behaves differently from a hole at the plate's midline (symmetry intent) when the plate grows. The discipline: fully define every sketch (no under-constrained geometry drifting at a click), reference datums and axes rather than face edges that may vanish, name features, and order the tree so machining-like operations come last. Model as if the next engineer will change the one dimension you least expect — because they will.

03 · From model to manufacture

The model is master; drawings, CAM paths, and 3-D prints are projections of it. Materials, mass properties, and revision metadata live on the model. For the maintenance engineer, the skill is bidirectional: read OEM models when you have them, and rebuild worn parts as clean parametric models when you don't — Lesson 3's field sketch becomes tonight's model and tomorrow's machined spare.

Closing. Parametric CAD is programming with geometry: fully defined sketches, functional references, an ordered tree — that is what separates a model from a liability.

Foundations

What this lesson assumes

  • Views and dimensioning (Lessons 2, 5) — sketch constraints are dimensions with intent.
  • Datum thinking (Lesson 7) — model references mirror measurement datums.

Glossary

  • Feature tree — the ordered recipe of operations.
  • Fully defined sketch — zero degrees of freedom; nothing drifts.
  • Design intent — the declared invariants under change.
  • Reference geometry — planes/axes/points created to anchor features stably.
  • Rebuild error — the tree failing after an upstream change; the tax on lazy references.

Examples and Quiz

Worked example 1 — intent, tested

Problem. A 100×60 plate has a ⌀9 hole. Model A dimensions the hole 20 from the left edge; Model B constrains it to the plate's vertical midline. The plate is later widened to 140. Find: where the hole lands in each model, and which intent suits (a) a clearance hole for a fixed stud, (b) a lifting hole.

Solution. (1) Model A: hole stays 20 from the left edge — absolute offset preserved. (2) Model B: hole rides to the new midline (70) — symmetry preserved. (3) Function decides: (a) the stud doesn't move when the plate grows → offset intent (A); (b) lifting wants balance → symmetry intent (B).

Answer: A: x=20; B: x=70; choose by function — (a) A, (b) B. Sanity check: neither model is "right" in the abstract; intent is a functional claim, which is why it must be chosen, not defaulted.

Worked example 2 — the fillet that killed the tree

Problem. A bracket model references a hole sketch to the edge created by a fillet. A revision deletes the fillet; forty downstream features error out. Find: the modeling fault and the repair strategy.

Solution. (1) Fault: sketch referenced consumable geometry (a fillet edge) instead of stable references (datum plane / base edge). (2) Repair: redefine the sketch onto the base face and datum axis, then re-run the tree; add reference planes for any future features. (3) Prevention: reference datums first, faces second, feature edges never — the same hierarchy inspection uses.

Answer: re-anchor to stable references; adopt datum-first modeling. Sanity check: the fix touches one sketch, not forty features — proof the fault was referential, not geometric.

Applied case — factory floor (Kuwait)

Vignette — the impeller wear ring, digitized. You arrive at a stores review where a pump's wear rings — obsolete, OEM defunct — are consuming six-week emergency machining cycles from a faded fax of a sketch. One evening of parametric modeling from a measured worn ring (Lesson 3's sketch, Lesson 6's fits restoring the design clearance) produces a clean model with the running clearance driven by one named parameter. The next order is a two-day local machining job with the drawing auto-generated from the model. The fax retires; the parameter ring_clearance is now plant knowledge instead of folklore.
