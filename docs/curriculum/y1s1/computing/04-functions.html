Functions

Functions as named, testable calculations. Arguments, returns, and docstrings that state units. Building a small library of personal engineering functions.

Lecture

After this lesson you can answer:
  • What belongs in a function's docstring for an engineering calculation?
  • Why do small, single-purpose functions beat one long script?

01 · Functions: calculations with names

A function packages a calculation behind a name and an interface: def shaft_power(q_m3s, head_m, eta=0.7): declares what goes in, return declares what comes out, and everything between is the calculation, written once and reused forever. The docstring — a quoted paragraph directly under the def — states what the function does AND its units, which turns the function into a self-documenting engineering tool: help(shaft_power) answers the reviewer before the reviewer asks. Default arguments (the eta=0.7) encode the usual case while keeping the unusual case one keyword away.

02 · Return, not print

A function that PRINTS its result has spent it; a function that RETURNS it hands the value to the caller for further use — feeding the next calculation, a comparison, a plot, a test. Printing is presentation, the script's outer layer; returning is plumbing, the composable interior. The distinction becomes visible the first time you try to use a printed result in arithmetic and find you hold None. Compose functions the way the chain rule composed rates: small functions, each doing one nameable thing, wired by their returns — \( \texttt{gap(c\_to\_k(90), c\_to\_k(30))} \) reads like the sentence it implements.

03 · Scope: what happens in a function stays there

Variables born inside a function are local: invisible outside, gone when the call ends. This is a feature — ten functions can each use x without collision — and a discipline: a function should receive what it needs as arguments and deliver what it produces as returns, never reaching sideways into the script's globals. Functions written that way are testable in isolation, which is exactly how the next lesson will test them.

04 · The function-design working method

Extract a function whenever a calculation is used twice, or once but nameable. Name it by what it returns (hydraulic_power_w), carry units in parameter names, and write the docstring FIRST — if you cannot state the contract in a sentence, the function is not yet one thing. Keep functions small enough to hand-verify: one formula, one decision, one transformation each. Then apply Lesson 1's rule at the function scale: every function meets a hand-checked case before it is trusted, and the check lives beside the function as next lesson's assertion. A personal library of such functions — built, checked, and reused across this course — is the deliverable this working method is quietly assembling.

Resist the two classic function pathologies while the habits are forming. The do-everything function — read the file, convert the units, compute the power, print the report — cannot be tested except by running the world; split it at every nameable seam and each piece becomes checkable alone. And the copy-paste twin — the same formula pasted in three places with one variable renamed — guarantees that a future correction reaches two of the three; extraction into one function makes the fix land everywhere by construction. Functions are the unit of reuse, of testing, and of trust: size them so all three work.

Examples and Quiz

Problem 1 of 8 · solve, then check

Build and trace the power function

Problem. Trace the call shaft_power(0.025, 25):

def shaft_power(q_m3s, head_m, eta=0.7):
    """Shaft power in W for water. q in m3/s, head in m."""
    p_hyd = 1000 * 9.81 * q_m3s * head_m
    return p_hyd / eta

Solution. (1) \( p_{hyd} = 1000 \times 9.81 \times 0.025 \times 25 = 6131\ \mathrm{W} \). (2) η defaults to 0.7: \( 6131/0.7 = 8759\ \mathrm{W} \). (3) Returns ≈8.76 kW (the value — nothing prints unless the caller prints).

Answer: ≈8759 W returned. Sanity check: the default carried the usual efficiency silently; a stiffer pump calls shaft_power(0.025, 25, eta=0.8) → 7664 W. One function now serves every pump problem this course has computed by hand — which is precisely the point of writing it.

Problem 2 of 8 · solve, then check

Three statistics, one return

Problem. Write a function returning the minimum, maximum, and mean of a list, and trace it on [3, 9, 6].

Solution. (1) def stats(xs): return min(xs), max(xs), sum(xs)/len(xs) (2) Trace: min 3, max 9, mean \( 18/3 = 6.0 \). (3) Caller unpacks: lo, hi, avg = stats(readings).

Answer: \( (3,\ 9,\ 6.0) \). Sanity check: the mean lies between min and max — the free ordering audit every statistics return should pass; and returning the tuple (rather than printing three lines) leaves the caller free to alarm on hi, trend avg, or ignore lo: plumbing, not presentation.

Problem 3 of 8 · solve, then check

Composed conversions

Problem. Given def c_to_k(t_c): return t_c + 273.15 and def gap(a, b): return a - b, trace gap(c_to_k(90), c_to_k(30)) and comment on the design.

Solution. (1) Inner calls first: \( 90 + 273.15 = 363.15 \); \( 30 + 273.15 = 303.15 \). (2) gap(363.15, 303.15) → 60.0 K.

Answer: 60.0 K. Sanity check: a temperature DIFFERENCE is identical in °C and K — the offsets cancel, and the composition proves it numerically. Two tiny, individually verifiable functions composed by returns produced a correct result neither prints: composition is how small trusted pieces build large trusted calculations.

Problem 4 of 8 · multiple choice

A function's docstring should state, at minimum:

The docstring is the function's datasheet: contract plus units, readable via help() without opening the source. Authorship and dates belong to version control, which tracks them automatically. 'Self-documenting code' covers WHAT lines do, never which unit a parameter expects — and unit ambiguity, as ever, is the error that costs the most and announces itself the least.

Problem 5 of 8 · multiple choice

A function that prints its result instead of returning it:

Printing sends characters to the screen and returns None — try arithmetic on it and the error arrives one line later, mystifying beginners. Returning hands the caller a value to compute with, test, or print AT THE EDGE of the script. The rule of thumb: functions return; the script's last lines print. Everything composable follows from it.

Problem 6 of 8 · multiple choice

A variable created inside a function is, outside the function:

Local scope: born at the call, gone at the return. This isolation is what lets ten functions use x without collision, and what makes each testable alone. Needing a function's inner value outside is the signal to RETURN it — reaching in (or leaking globals out) couples everything to everything, and coupled scripts break in sympathy.

Problem 7 of 8 · multiple choice

With def f(x): return x * x, the value of f(f(2)) is:

Inside-out: f(2) = 4, then f(4) = 16. Composition evaluates inner calls first — exactly the chain-rule reading order from the mathematics course. Answering 8 computes \( 2^3 \), a different composition; 4 stops after one call. Tracing nested calls inner-first is a two-second discipline that scales to arbitrarily deep pipelines.

Problem 8 of 8 · multiple choice

In def shaft_power(q_m3s, head_m, eta=0.7), the eta=0.7 means:

A default argument encodes the usual case without forbidding the unusual: shaft_power(q, h) uses 0.7; shaft_power(q, h, eta=0.82) overrides it. Defaults document typical practice right in the signature — and keep call sites short where the typical applies. The pattern recurs across every engineering library you will ever import.
