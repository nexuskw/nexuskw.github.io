Lists, dictionaries, and files

Collections for measurement sets and equipment records. Reading and writing CSV files — the lingua franca of plant data exports. Cleaning real, messy data defensively.

01 · Lists: ordered data with a name

Engineering data arrives in sequences — readings over a route, temperatures over a shift — and the list is Python's sequence: readings = [2.1, 4.5, 3.2], ordered, indexable, growable. Indexing starts at ZERO — readings[0] is the first item — and negative indices count from the end, readings[-1] being the latest reading, an idiom worth owning immediately. Slices carve subranges (readings[1:3] yields items 1 and 2 — the stop index is excluded, mirroring range), append grows the list in place, and the built-ins len, sum, min, max answer the first questions any dataset is asked. The list comprehension compresses the transform loop to a phrase — [x * 0.746 for x in hp_values] — and reads like the sentence it implements.

02 · Dictionaries: records with named fields

Where lists order items, dictionaries name them: pump = {"tag": "P-101", "power_kw": 11.5} is an equipment record whose fields are looked up by key — pump["power_kw"] — not by position. Adding a field is one assignment; asking whether one exists is "tag" in pump. The two structures compose into the universal data shape of plant work: a LIST of DICTIONARIES — one dict per equipment item, one list per fleet — which is exactly what a CSV file of records becomes the moment Python reads it.

03 · Files: where real data lives

The CSV — comma-separated values, one record per line — is the lingua franca of every historian export, test-bench log, and maintenance system dump. Reading one is three idioms stacked: iterate the file line by line, split(",") each line into fields, and convert numeric fields with float() at the border, exactly as Lesson 2 demanded. The with open(...) as f: form opens the file and guarantees it closes. Everything after the read is Lessons 3–4: loops, accumulators, and functions, now fed by real data instead of typed constants.

04 · The data-structure working method

Choose the structure by the question the data answers. Ordered measurements → list; named fields of one thing → dict; a fleet of records → list of dicts; and when you catch yourself tracking parallel lists (tags in one, powers in another), that is the signal the records want to be dicts. Convert types at the file border, never downstream. Trust the built-ins before writing loops — sum, max, len are tested code you did not have to write. And hand-trace list operations on a three-item example before applying them to thirty thousand rows, exactly as the loop lesson traced its accumulators: the small case proves the pattern, the large case merely repeats it.

One caution as datasets grow: a structure choice made casually at three records becomes architecture at thirty thousand. The list-of-dicts shape scales gracefully because every operation this course teaches — loops, comprehensions, filters, summaries — walks it unchanged; ad-hoc shapes (nested lists of mixed meaning, parallel lists) accumulate special cases until the script is mostly exceptions. Choose the boring, regular shape early and the analysis lessons ahead inherit clean footing.

Examples and Quiz

Work every problem on paper first. Solved examples reveal their full solution; for the multiple-choice questions, select your answers and press Submit to see what was right, what was not, and why.

Problem 1 of 8 · solve, then check

The route list, operated

Problem. Trace: start with readings = [2.1, 4.5, 3.2]; append 5.0; report the last element, the maximum, and the length.

Solution. (1) After readings.append(5.0): [2.1, 4.5, 3.2, 5.0]. (2) readings[-1] → 5.0 (the newest); max(readings) → 5.0; len(readings) → 4.

Answer: 5.0, 5.0, 4. Sanity check: the newest reading happening to BE the maximum is the vibration-route bad news pattern — and readings[-1] == max(readings) is exactly the one-line trend alarm a later script will build from these three idioms. Small operations, real diagnostics.

Problem 2 of 8 · solve, then check

An equipment record that grows

Problem. Build the record pump = {"tag": "P-101", "power_kw": 11.5}; add a field "status": "running"; read the power; and show the fleet shape for two pumps.

Solution. (1) Add: pump["status"] = "running" — dicts grow by assignment. (2) Read: pump["power_kw"] → 11.5. (3) Fleet: fleet = [{"tag": "P-101", "power_kw": 11.5}, {"tag": "P-102", "power_kw": 7.5}] — a list of dicts, iterated as for pump in fleet:.

Answer: field added, 11.5 read, fleet as list-of-dicts. Sanity check: the alternative — parallel lists of tags and powers — breaks the moment one list is sorted or filtered without the other; the record keeps each pump's facts fused. This shape IS the maintenance database row, and recognizing it is the lesson.

Problem 3 of 8 · solve, then check

Reading the export

Problem. A file contains two lines: P-101,11.5 and P-102,7.5. Script the total power and trace it.

Solution. (1) total = 0.0 with open("pumps.csv") as f: for line in f: fields = line.split(",") total += float(fields[1]) (2) Trace: 0.0 + 11.5 = 11.5; 11.5 + 7.5 = 19.0.

Answer: 19.0 kW total. Sanity check: every piece is a prior lesson — the with-open border, the split into fields, the float() conversion (without which += would crash on a string), the accumulator. File processing is not a new skill; it is the old skills fed by a file, which is why the course built them first.

Problem 4 of 8 · multiple choice

For a = [10, 20, 30, 40], the value of a[1] is:

Zero-based indexing: a[0] is 10, so a[1] is 20 — the SECOND item. The 1-counting instinct from everyday life is the source of most beginner index errors, and range(n)'s 0-to-n−1 convention (Lesson 3) was designed to match this exactly: a[i] for i in range(len(a)) touches every element with no off-by-one anywhere.

Problem 5 of 8 · multiple choice

The idiomatic way to read the LAST element of list a is:

Negative indices count from the end: a[-1] is the last, a[-2] the one before. a[len(a)] is one PAST the end — an IndexError, and the classic fencepost slip (the valid last index is len(a)−1). The other forms belong to other languages. For time-ordered data, a[-1] is 'the latest reading' — an idiom used daily.

Problem 6 of 8 · multiple choice

[x * 2 for x in [1, 2, 3]] evaluates to:

The comprehension applies the expression to each element: doubles, collected in order. Repetition ([1,2,3,1,2,3]) is what [1,2,3] * 2 does — the list-level operator, a different animal; squares would need x ** 2. Comprehensions are the transform loop compressed to a readable phrase, and they appear in nearly every data script from here on.

Problem 7 of 8 · multiple choice

Field values pulled from a CSV line by split(",") arrive as:

Files are text; split returns strings, whatever they resemble — and "11.5" + "7.5" concatenates to "11.57.5" rather than adding. CSV declares no types; nothing converts automatically. The border-conversion rule of Lesson 2 exists precisely for this moment: float() each numeric field at the read, and the interior of the script stays safely numeric.

Problem 8 of 8 · multiple choice

Tracking pump tags in one list and their powers in a parallel list becomes fragile because:

Parallel lists are coupled by nothing but hope: one sort, one filter, one insertion applied to only one of them, and every tag points at the wrong power — silently. The list-of-dicts shape fuses each record's fields so they travel together through any operation. Memory and speed are irrelevant at this scale; integrity is the argument, and it is decisive.
